# Creating Pull Requests

- Follow the PR-scope instructions in `CONTRIBUTING.md`.
- For any change that adds a feature or fixes a problem, add a `CHANGELOG.md` entry that concisely describes the change.
- Put details/background in the commit message, NOT the changelog.
- Match the existing `CHANGELOG.md` format; add the entry to the correct section.
